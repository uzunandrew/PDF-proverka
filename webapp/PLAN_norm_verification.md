# План: Автоматическая верификация нормативных ссылок

## Проблема
ИИ при формировании замечаний ссылается на нормативные документы (СП, ГОСТ, ПУЭ),
но может ошибаться: указать устаревшую редакцию, отменённый документ, или неверный пункт.
Сейчас проверка зависит от "настроения" модели — нет гарантии что WebSearch будет вызван.

## Решение
Добавить **обязательный этап верификации** после формирования замечаний (03_findings.json).
Для каждой нормативной ссылки — автоматический поиск в интернете через docs.cntd.ru.
Если норма устарела — пересмотр замечания с учётом актуальной нормы.

## Новый конвейер (было 4 этапа → стало 5)

```
[00] Init → [01] Текст → [02] Тайлы → [03] Свод → [03.5] ВЕРИФИКАЦИЯ НОРМ → [04] Отчёт
```

## Что создаём

### 1. Скрипт `verify_norms.py` (Python, ~200 строк)

**Вход:** `_output/03_findings.json`
**Выход:** `_output/03a_norms_verified.json`

Алгоритм:
1. Прочитать `03_findings.json`
2. Извлечь ВСЕ нормативные ссылки из поля `norm` каждого finding
3. Для каждой уникальной нормы:
   a. Сначала проверить в `norms_reference.md` (локальный кэш)
   b. Если нормы нет в кэше — отправить запрос Claude CLI с инструкцией:
      "Выполни WebSearch на docs.cntd.ru: [название нормы]. Сообщи статус: действует/отменён/заменён, актуальная редакция."
   c. Записать результат
4. Сформировать отчёт верификации:
   - Для каждого finding: норма OK / норма устарела / норма отменена
   - Если норма устарела — указать актуальную замену
5. Если есть проблемные нормы — запустить ещё одну Claude-сессию:
   "Пересмотри замечания F-002, F-003 с учётом актуальных норм: [список замен]"
6. Записать `03a_norms_verified.json` — копия findings с исправленными нормами + лог верификации

### 2. Шаблон задачи `.claude/norm_verify_task.md`

Шаблон для Claude CLI сессии верификации:
- На входе: список норм для проверки
- Инструкция: для каждой нормы выполнить WebSearch на docs.cntd.ru
- На выходе: JSON с результатами проверки

### 3. Шаблон задачи `.claude/norm_fix_task.md`

Шаблон для пересмотра замечаний:
- На входе: замечания с устаревшими нормами + актуальные замены
- Инструкция: пересмотреть формулировки с учётом новых норм
- На выходе: исправленные findings

### 4. Интеграция в pipeline_service.py

- Новый AuditStage: `NORM_VERIFY = "norm_verify"`
- Новый метод: `_run_norm_verification(job)`
- В `_run_full_pipeline`: вставить между этапом 3 (main_audit) и этапом 4 (excel)

### 5. Интеграция в webapp (фронтенд)

- Новый шаг пайплайна "03a" между "03" и "04"
- Кнопка "Верификация норм" в управлении аудитом
- Отдельная таблица/секция в замечаниях: "Нормы проверены ✓ / Нормы исправлены ⚠️"

## Формат 03a_norms_verified.json

```json
{
  "meta": {
    "project_id": "...",
    "verification_date": "ISO",
    "total_norms_checked": 12,
    "norms_ok": 10,
    "norms_outdated": 1,
    "norms_replaced": 1,
    "findings_revised": 2
  },
  "norm_checks": [
    {
      "norm_original": "СП 256.1325800.2016 (ред. изм. 1-5)",
      "status": "outdated_edition",
      "current_version": "СП 256.1325800.2016 (ред. 29.01.2024, изм. 1-7)",
      "source": "docs.cntd.ru",
      "check_date": "ISO",
      "affected_findings": ["F-003"]
    }
  ],
  "findings": [
    // Полная копия findings из 03_findings.json
    // Если норма была исправлена — поле norm обновлено
    // + добавлено поле "norm_verified": true/false
    // + добавлено поле "norm_revision_note": "Изменено: было X → стало Y"
  ]
}
```

## Порядок реализации

1. `verify_norms.py` — ядро верификации
2. `.claude/norm_verify_task.md` — шаблон поиска
3. `.claude/norm_fix_task.md` — шаблон пересмотра
4. `webapp/models/audit.py` — добавить NORM_VERIFY в AuditStage
5. `webapp/config.py` — добавить пути к новым шаблонам
6. `webapp/services/claude_runner.py` — добавить run_norm_verify() и run_norm_fix()
7. `webapp/services/pipeline_service.py` — вставить этап верификации
8. `webapp/routers/audit.py` — добавить эндпоинт /verify-norms
9. `webapp/static/` — обновить фронтенд (пайплайн 5 шагов, кнопка, статус)
