## Что искать на чертежах

### Однолинейные схемы
- Номиналы автоматических выключателей — соответствие расчёту нагрузок
- Наличие УЗО/УЗДП на нужных группах (розеточные 30 мА, ванные 10 мА)
- Трансформаторы тока (ТТ): номинал, класс точности, вторичный ток
- АВР для потребителей I категории и СПЗ
- Учёт электроэнергии (счётчики, АСКУЭ)
- Соответствие кабелей на схеме спецификации
- Селективность защиты (номиналы вышестоящий > нижестоящий)

### Схемы щитов
- Совпадение групп со спецификацией
- Сечения кабелей соответствуют нагрузкам и защитным аппаратам
- Дубли групп (одинаковые обозначения)
- Наличие PE-шины
- Номиналы дифавтоматов и УЗО

### Планы (поэтажные, паркинга)
- Цвета кабелей соответствуют системам (синий=норм, красный=ОКЛ)
- Все щиты с плана присутствуют в спецификации
- Трассы ОКЛ не проходят через незащищённые зоны
- Расположение оборудования (ЩУ, ЩД, ЩР)

### Узлы ввода, детали
- Способ прокладки кабелей ввода
- Наличие защиты от механических повреждений
- Сечения вводных кабелей
- Заземление, уравнивание потенциалов

### Спецификации, таблицы
- Марки кабелей по ГОСТ
- Объёмы (метражи, количество)
- Соответствие позиций чертежей и таблиц

## Правила

1. **Прочитай КАЖДЫЙ тайл** — не пропускай ни один
2. Для каждого тайла ОБЯЗАТЕЛЬНО запиши summary и key_values_read
3. Если тайл пустой или содержит только штамп — запиши "Штамп / служебная информация"
4. findings может быть пустым [] если замечаний нет — это нормально
5. Нумерация: G-001, G-002, ... (в пределах этого пакета, при слиянии перенумеруется)
6. severity — ТОЛЬКО одно из 5 значений: КРИТИЧЕСКОЕ, ЭКОНОМИЧЕСКОЕ, ЭКСПЛУАТАЦИОННОЕ, РЕКОМЕНДАТЕЛЬНОЕ, ПРОВЕРИТЬ ПО СМЕЖНЫМ
7. Пиши JSON через инструмент Write — НЕ выводи в чат
8. После записи JSON выведи краткий итог: сколько тайлов, сколько находок
