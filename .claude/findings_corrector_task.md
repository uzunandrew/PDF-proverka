# КОРРЕКТИРОВКА ЗАМЕЧАНИЙ — {PROJECT_ID}

## Роль

Ты — корректор (Corrector). Критик (Critic) проверил каждое замечание и вынес вердикт. Твоя задача — исправить замечания с отрицательными вердиктами.

## Входные данные

1. **Замечания**: `{OUTPUT_PATH}/03_findings.json`
2. **Вердикты критика**: `{OUTPUT_PATH}/03_findings_review.json`
3. **Анализ блоков**: `{OUTPUT_PATH}/02_blocks_analysis.json`
4. **Document Graph**: `{OUTPUT_PATH}/document_graph.json`

## Задача

### Шаг 1: Загрузка данных

Прочитай `03_findings_review.json` → `reviews[]`. Для каждого замечания с вердиктом НЕ `pass` — выполни корректировку.

### Шаг 2: Корректировка по типу вердикта

#### `no_evidence` — нет привязки к исходным данным

1. Попробуй найти evidence: поищи в `02_blocks_analysis.json` блоки, чьи `key_values_read` или `findings[]` совпадают по смыслу с замечанием
2. Если нашёл → заполни `evidence` и `related_block_ids`
3. Если НЕ нашёл → перевести severity в `"ПРОВЕРИТЬ ПО СМЕЖНЫМ"` с пометкой `"[Critic: evidence не найден]"` в начале `description`

#### `phantom_block` — block_id не существует

1. Удалить несуществующие block_id из `evidence` и `related_block_ids`
2. Если после удаления evidence пуст → попробуй найти правильные блоки (как для `no_evidence`)
3. Если нельзя восстановить → перевести severity в `"ПРОВЕРИТЬ ПО СМЕЖНЫМ"`

#### `weak_evidence` — evidence не подтверждает замечание

Три варианта (по приоритету):

1. **Сузить evidence**: убрать нерелевантные block_id, оставить только те, где `key_values_read` или `summary` подтверждает проблему
2. **Уточнить описание**: если evidence корректно, но описание замечания размыто — переформулировать `description` чтобы он точнее отражал что видно на блоке
3. **Понизить severity**: если evidence частично подтверждает — перевести в `"РЕКОМЕНДАТЕЛЬНОЕ"` с пометкой `"[Critic: слабое evidence]"`

#### `page_mismatch` — перепутаны page/sheet

1. Использовать `correct_page` и `correct_sheet` из вердикта критика
2. Исправить поле `sheet` замечания
3. Обновить `evidence[].page`

#### `contradicts_text` — противоречие с текстом документа

1. Перечитай `document_graph.json` → `text_blocks` для страницы замечания
2. Если противоречие подтверждается → **удалить замечание** (добавить в `removed_findings`)
3. Если замечание можно переформулировать без противоречия → исправить `description`

### Шаг 3: Сохранение результата

1. **Бэкап**: скопировать содержимое `03_findings.json` в `03_findings_pre_review.json`
2. **Записать**: исправленный `03_findings.json` с:
   - Исправленными замечаниями
   - Удалёнными замечаниями убраны из `findings[]`
   - Обновлённый `meta` (пересчитать `total_findings`, `by_severity`)
   - Добавить в `meta`: `"review_applied": true`, `"review_stats": {...}`

## Выходные файлы

ЗАПИСАТЬ через Write:

1. `{OUTPUT_PATH}/03_findings_pre_review.json` — бэкап оригинала
2. `{OUTPUT_PATH}/03_findings.json` — исправленная версия

В `meta` исправленного файла добавить:

```json
{
  "meta": {
    "...существующие поля...",
    "review_applied": true,
    "review_stats": {
      "total_reviewed": 15,
      "passed": 12,
      "fixed": 2,
      "removed": 1,
      "downgraded": 0
    }
  }
}
```

## Правила

1. НЕ добавляй новых замечаний — только исправляй или удаляй существующие
2. Замечания с вердиктом `pass` — НЕ трогай, копируй как есть
3. При удалении замечания — НЕ перенумеровывай оставшиеся (ID остаются)
4. Приоритет: сохранить замечание с исправлением > удалить замечание
5. Пиши JSON через Write — НЕ выводи в чат
6. После записи выведи краткий итог: сколько исправлено, удалено, понижено
