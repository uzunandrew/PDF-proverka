## Задача

### Шаг 1: Загрузка данных

Прочитай `optimization_review.json` → `reviews[]`. Для каждого предложения с вердиктом НЕ `pass` — выполни корректировку.

### Шаг 2: Корректировка по типу вердикта

#### `vendor_violation` — производитель не из вендор-листа

1. Заменить предложенного производителя на ближайший аналог из вендор-листа выше
2. Если подходящего аналога нет → удалить предложение (в `removed_items`)
3. Пересчитать `savings_pct` если замена влияет на цену

#### `conflicts_with_finding` — конфликт с замечанием аудита

1. Если позиция имеет КРИТИЧЕСКОЕ замечание → **удалить** предложение оптимизации (нельзя оптимизировать то, что нарушает нормы)
2. Если ЭКОНОМИЧЕСКОЕ → изменить `status` на `"обязательное исправление"` и переформулировать `proposed` с учётом замечания
3. Добавить в `risks` ссылку на замечание аудита

#### `unrealistic_savings` — завышенная экономия

1. Если `savings_basis` = `"не определено"` → установить `savings_pct` = `0`
2. Если `savings_basis` = `"экспертная оценка"` и `savings_pct > 30` → снизить до реалистичного уровня (10-20%)
3. Обновить `savings_basis` на `"экспертная оценка"` если был `"расчёт"` без обоснования

#### `no_traceability` — нет привязки к документу

1. Поищи в MD-файле упомянутую позицию/оборудование
2. Если нашёл → заполни `spec_items`, `page`, `sheet`, `section`
3. Если НЕ нашёл → удалить предложение (необоснованное)

#### `wrong_page` — неверная страница

1. Найди правильную страницу через MD-файл или `document_graph.json`
2. Исправь `page`, `sheet`, `section`

#### `too_vague` — слишком общее предложение

1. Конкретизировать `current` и `proposed` — добавить конкретные марки, размеры, параметры из MD-файла
2. Заполнить `spec_items` конкретными позициями
3. Если конкретизировать невозможно (нет данных в документе) → удалить предложение

#### `technical_issue` — техническая ошибка

1. Если ошибка исправима (неверные параметры) → исправить `proposed` и `norm`
2. Если предложение нарушает нормы → удалить
3. Если неверный `type` → исправить на правильный

### Шаг 3: Сохранение результата

1. **Бэкап**: скопировать содержимое `optimization.json` в `optimization_pre_review.json`
2. **Записать**: исправленный `optimization.json` с:
   - Исправленными предложениями
   - Удалённые предложения убраны из `items[]`
   - Обновлённый `meta` (пересчитать `total_items`, `by_type`, `estimated_savings_pct`)
   - Добавить в `meta`: `"review_applied": true`, `"review_stats": {...}`

## Выходные файлы

ЗАПИСАТЬ через Write:

1. `{OUTPUT_PATH}/optimization_pre_review.json` — бэкап оригинала
2. `{OUTPUT_PATH}/optimization.json` — исправленная версия

В `meta` исправленного файла добавить:

```json
{
  "meta": {
    "...существующие поля...",
    "review_applied": true,
    "review_stats": {
      "total_reviewed": 10,
      "passed": 7,
      "fixed": 2,
      "removed": 1
    }
  }
}
```

## Правила

1. НЕ добавляй новых предложений — только исправляй или удаляй существующие
2. Предложения с вердиктом `pass` — НЕ трогай, копируй как есть
3. При удалении предложения — НЕ перенумеровывай оставшиеся (ID остаются)
4. Замены производителей — ТОЛЬКО из вендор-листа выше
5. Приоритет: сохранить предложение с исправлением > удалить предложение
6. Пиши JSON через Write — НЕ выводи в чат
7. После записи выведи краткий итог: сколько исправлено, удалено
